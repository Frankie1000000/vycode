React.js

React is a JavaScript library for building user interfaces, particularly single-page applications where data changes over time. It was developed by Facebook and is now maintained by Facebook and the community.

Key Features:

  • Component-based architecture
  • Virtual DOM for efficient updates
  • Unidirectional data flow
  • JSX syntax for writing components
  • Hooks for state and lifecycle management

Example: Simple Counter Component

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

// Render the component
ReactDOM.render(<Counter />, document.getElementById('root'));

Vue.js

Vue.js is a progressive JavaScript framework for building user interfaces. Unlike other monolithic frameworks, Vue is designed from the ground up to be incrementally adoptable.

Key Features:

  • Reactive data binding
  • Component-based architecture
  • Simple and flexible API
  • Directives for DOM manipulation
  • Transition effects system

Example: Simple Vue App

<div id="app">
  {{ message }}
  <button v-on:click="reverseMessage">Reverse Message</button>
</div>

<script>
new Vue({
  el: '#app',
  data: {
    message: 'Hello Vue.js!'
  },
  methods: {
    reverseMessage: function () {
      this.message = this.message.split('').reverse().join('')
    }
  }
})
</script>

Angular

Angular is a platform and framework for building single-page client applications using HTML and TypeScript. Angular is written in TypeScript and implements core and optional functionality as a set of TypeScript libraries.

Key Features:

  • TypeScript-based
  • Dependency injection
  • Two-way data binding
  • Modular architecture
  • Comprehensive routing solution

Example: Simple Component

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: `
    <h1>{{title}}</h1>
    <button (click)="onClick()">Click me!</button>
    <p>{{message}}</p>
  `
})
export class AppComponent {
  title = 'My Angular App';
  message = 'Hello Angular!';

  onClick() {
    this.message = 'Button was clicked!';
  }
}

Svelte

Svelte is a radical new approach to building user interfaces. Whereas traditional frameworks like React and Vue do the bulk of their work in the browser, Svelte shifts that work into a compile step that happens when you build your app.

Key Features:

  • No virtual DOM
  • Truly reactive
  • Compiles to tiny, framework-less vanilla JS
  • Simple syntax with minimal boilerplate
  • Built-in state management

Example: Svelte Component

<script>
  let count = 0;
  
  function increment() {
    count += 1;
  }
</script>

<button on:click={increment}>
  Clicked {count} {count === 1 ? 'time' : 'times'}
</button>

State Management (Redux, Vuex)

State management libraries help manage application state in a predictable way. Redux is commonly used with React, while Vuex is the official state management library for Vue.

Key Concepts:

  • Single source of truth
  • State is read-only
  • Changes are made with pure functions
  • Centralized state management
  • Time-travel debugging

Example: Redux Store Setup

import { createStore } from 'redux';

// Reducer function
function counter(state = 0, action) {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    case 'DECREMENT':
      return state - 1;
    default:
      return state;
  }
}

// Create a Redux store
let store = createStore(counter);

// Subscribe to store updates
store.subscribe(() => {
  console.log(store.getState());
});

// Dispatch actions
store.dispatch({ type: 'INCREMENT' });
store.dispatch({ type: 'INCREMENT' });
store.dispatch({ type: 'DECREMENT' });

Routing (React Router, Vue Router)

Routing libraries enable navigation between different components/views in a single-page application, changing the browser URL, and keeping UI in sync with the URL.

Example: React Router

import { BrowserRouter as Router, Route, Link } from 'react-router-dom';

function Home() {
  return <h2>Home</h2>;
}

function About() {
  return <h2>About</h2>;
}

function App() {
  return (
    <Router>
      <div>
        <nav>
          <ul>
            <li><Link to="/">Home</Link></li>
            <li><Link to="/about">About</Link></li>
          </ul>
        </nav>

        <Route path="/" exact component={Home} />
        <Route path="/about" component={About} />
      </div>
    </Router>
  );
}

Server-Side Rendering (SSR)

Server-Side Rendering generates the complete HTML for a page on the server in response to navigation. This improves SEO and initial load performance.

Benefits of SSR:

  • Improved SEO as search engines can crawl content
  • Faster initial page load
  • Better performance on low-powered devices
  • Social media sharing previews work correctly

Example: Next.js (React SSR)

// pages/index.js
import React from 'react';

export default function HomePage({ data }) {
  return (
    <div>
      <h1>Server-Side Rendered Page</h1>
      <p>Data fetched on server: {data}</p>
    </div>
  );
}

// This gets called on every request
export async function getServerSideProps() {
  // Fetch data from external API
  const res = await fetch('https://api.example.com/data');
  const data = await res.json();

  // Pass data to the page via props
  return { props: { data } };
}

Static Site Generation (SSG)

Static Site Generation pre-renders pages at build time. The HTML is generated once and served to all users, providing excellent performance.

Benefits of SSG:

  • Exceptional performance with pre-built HTML
  • Reduced server load and cost
  • Better security with no server-side processing
  • Easy to deploy on CDNs

Example: Next.js Static Generation

// pages/posts/[id].js
import React from 'react';

export default function Post({ post }) {
  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  );
}

// This function gets called at build time
export async function getStaticPaths() {
  // Fetch list of posts from API
  const res = await fetch('https://api.example.com/posts');
  const posts = await res.json();

  // Get the paths we want to pre-render
  const paths = posts.map(post => ({
    params: { id: post.id.toString() },
  }));

  // Pre-render only these paths at build time
  return { paths, fallback: false };
}

// This also gets called at build time
export async function getStaticProps({ params }) {
  // Fetch single post detail
  const res = await fetch(`https://api.example.com/posts/${params.id}`);
  const post = await res.json();

  // Pass post data to the page via props
  return { props: { post } };
}

Utility Libraries

Utility libraries provide helper functions for common programming tasks, making development more efficient and code more consistent.

Popular Utility Libraries:

  • Lodash - JavaScript utility library
  • date-fns - Date manipulation library
  • Axios - HTTP client
  • Immer - Immutable state updates
  • Classnames - Conditionally joining classNames

Example: Lodash Utility Functions

import _ from 'lodash';

// Array manipulation
const numbers = [1, 2, 3, 4, 5];
const doubled = _.map(numbers, n => n * 2);
console.log('Doubled numbers:', doubled);

// Object manipulation
const user = { name: 'John', age: 30, city: 'New York' };
const picked = _.pick(user, ['name', 'age']);
console.log('Picked properties:', picked);

// Debouncing a function
const search = _.debounce((query) => {
  console.log('Searching for:', query);
}, 300);

// Call the debounced function
search('react');
search('vue');
search('angular'); // Only this call will execute after 300ms

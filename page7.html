Window Object

The Window object represents a window containing a DOM document. It is the global object in browser JavaScript and provides methods to control the browser window, access the document, and handle events.

The Window interface is home to a variety of functions, namespaces, objects, and constructors which are not necessarily directly associated with the concept of a user interface window.

Example: Get Window Dimensions

// Get viewport dimensions
const width = window.innerWidth;
const height = window.innerHeight;

// Get screen dimensions
const screenWidth = window.screen.width;
const screenHeight = window.screen.height;

// Display the information
console.log(`Viewport: ${width}x${height}`);
console.log(`Screen: ${screenWidth}x${screenHeight}`);

// Open a new window
const newWindow = window.open('', '_blank', 'width=400,height=300');
newWindow.document.write('<h1>Hello from a new window!</h1>');

Document Object

The Document interface represents any web page loaded in the browser and serves as an entry point into the web page's content, which is the DOM tree.

The DOM tree includes elements such as <body> and <table>, among many others. It provides functionality to manipulate the document structure, style, and content.

Example: Document Manipulation

// Create a new element
const newDiv = document.createElement('div');
newDiv.textContent = 'This is a new div element';
newDiv.classList.add('highlight');

// Append to body
document.body.appendChild(newDiv);

// Select elements
const firstParagraph = document.querySelector('p');
const allDivs = document.querySelectorAll('div');

// Modify element style
firstParagraph.style.color = 'blue';
firstParagraph.style.fontSize = '20px';

console.log(`Number of div elements: ${allDivs.length}`);

Timers API

The WindowOrWorkerGlobalScope mixin provides the setTimeout() and setInterval() methods, which can be used to schedule code execution after a delay or at repeated intervals.

These methods return an ID that can be used to cancel the scheduled execution using clearTimeout() or clearInterval().

Example: Using Timers

// Set a timeout to execute after 2 seconds
const timeoutId = setTimeout(() => {
    console.log('This message appears after 2 seconds');
}, 2000);

// Set an interval to execute every second
let counter = 0;
const intervalId = setInterval(() => {
    counter++;
    console.log(`Interval count: ${counter}`);
    
    // Stop after 5 counts
    if (counter >= 5) {
        clearInterval(intervalId);
        console.log('Interval stopped');
    }
}, 1000);

// Clear a timeout (uncomment to test)
// clearTimeout(timeoutId);

LocalStorage & SessionStorage

The Web Storage API provides mechanisms for browsers to store key-value pairs in a much more intuitive fashion than using cookies.

localStorage persists data beyond the current session, even when the browser is closed and reopened.

sessionStorage maintains data only for the current session (until the browser is closed).

Example: Using localStorage

// Store data
localStorage.setItem('username', 'JohnDoe');
localStorage.setItem('theme', 'dark');

// Retrieve data
const username = localStorage.getItem('username');
const theme = localStorage.getItem('theme');

// Remove item
// localStorage.removeItem('theme');

// Clear all
// localStorage.clear();

console.log(`Username: ${username}, Theme: ${theme}`);

// Using sessionStorage (same API)
sessionStorage.setItem('sessionData', 'This will be lost when the session ends');

Geolocation API

The Geolocation API allows the user to provide their location to web applications if they so desire. For privacy reasons, the user is asked for permission to report location information.

This API is available through the navigator.geolocation object and provides methods to get the current position, watch for position changes, and clear watches.

Example: Get Current Location

function getLocation() {
  if (navigator.geolocation) {
    navigator.geolocation.getCurrentPosition(
      (position) => {
        const latitude = position.coords.latitude;
        const longitude = position.coords.longitude;
        const accuracy = position.coords.accuracy;
        
        console.log(`Latitude: ${latitude}, Longitude: ${longitude}`);
        console.log(`Accuracy: ${accuracy} meters`);
      },
      (error) => {
        console.error(`Error getting location: ${error.message}`);
      },
      {
        enableHighAccuracy: true,
        timeout: 5000,
        maximumAge: 0
      }
    );
  } else {
    console.error("Geolocation is not supported by this browser.");
  }
}

Canvas API

The Canvas API provides a means for drawing graphics via JavaScript and the HTML <canvas> element. It can be used for animation, game graphics, data visualization, photo manipulation, and real-time video processing.

The Canvas API largely focuses on 2D graphics. The WebGL API, which also uses the <canvas> element, draws hardware-accelerated 2D and 3D graphics.

Example: Basic Canvas Drawing

const canvas = document.createElement('canvas');
canvas.width = 300;
canvas.height = 200;
canvas.style.border = '1px solid black';

const ctx = canvas.getContext('2d');

// Draw rectangle
ctx.fillStyle = 'rgb(200, 0, 0)';
ctx.fillRect(10, 10, 50, 50);

// Draw circle
ctx.fillStyle = 'rgba(0, 0, 200, 0.5)';
ctx.beginPath();
ctx.arc(100, 35, 25, 0, Math.PI * 2, true);
ctx.fill();

// Draw text
ctx.font = '20px Arial';
ctx.fillStyle = 'black';
ctx.fillText('Hello Canvas', 70, 100);

// Output the canvas
document.getElementById('canvas-output').appendChild(canvas);

History API

The History API provides access to the browser's session history through the history object. It lets you manipulate the browser session history, that is the pages visited in the tab or frame that the current page is loaded in.

This is especially useful for single-page applications (SPAs) that need to manage navigation state without full page reloads.

Example: History Manipulation

// Get number of history entries
const historyLength = history.length;
console.log(`History length: ${historyLength}`);

// Navigate forward and backward
// history.back();    // Go to the previous page in history
// history.forward(); // Go to the next page in history
// history.go(-2);    // Go back two pages

// Add a new state to history
const state = { page: "settings", user: "john" };
history.pushState(state, "Settings", "/settings");

// Replace current state
// history.replaceState(state, "Settings", "/settings");

// Listen for popstate events (when user navigates through history)
window.addEventListener('popstate', (event) => {
  console.log('Location changed to: ' + document.location.href);
  console.log('State: ', event.state);
});

Drag and Drop API

The HTML Drag and Drop API enables applications to use drag-and-drop features in browsers. It allows users to click and hold the mouse button down over an element, drag it to another location, and release the mouse button to drop the element there.

The API defines events for drag and drop operations, data transfer mechanisms, and provides a way to customize the drag image and feedback.

Example: Basic Drag and Drop

// Make an element draggable
function makeDraggable(element) {
    element.draggable = true;
    
    element.addEventListener('dragstart', (e) => {
        e.dataTransfer.setData('text/plain', element.id);
        element.classList.add('dragging');
    });
    
    element.addEventListener('dragend', () => {
        element.classList.remove('dragging');
    });
}

// Make an element a drop target
function makeDropTarget(element) {
    element.addEventListener('dragover', (e) => {
        e.preventDefault(); // Necessary to allow dropping
        element.classList.add('drag-over');
    });
    
    element.addEventListener('dragleave', () => {
        element.classList.remove('drag-over');
    });
    
    element.addEventListener('drop', (e) => {
        e.preventDefault();
        element.classList.remove('drag-over');
        
        const id = e.dataTransfer.getData('text/plain');
        const draggable = document.getElementById(id);
        
        // Move the element to the drop target
        element.appendChild(draggable);
        
        console.log(`Dropped ${id} into drop zone`);
    });
}

// Initialize drag and drop functionality
document.addEventListener('DOMContentLoaded', () => {
    const draggableElements = document.querySelectorAll('.draggable');
    const dropTargets = document.querySelectorAll('.drop-target');
    
    draggableElements.forEach(makeDraggable);
    dropTargets.forEach(makeDropTarget);
});

Notifications API

The Notifications API allows web pages to display system notifications to the user. These appear outside the browser context, making them visible even when the user is not actively using the web page.

Notifications require user permission, which should be requested in response to a user gesture to ensure good user experience.

Example: Displaying a Notification

// Request permission for notifications
async function requestNotificationPermission() {
    if (!('Notification' in window)) {
        console.error('This browser does not support notifications');
        return false;
    }
    
    if (Notification.permission === 'granted') {
        return true;
    }
    
    const permission = await Notification.requestPermission();
    return permission === 'granted';
}

// Display a notification
function showNotification(title, options) {
    if (Notification.permission !== 'granted') {
        console.error('Notification permission not granted');
        return;
    }
    
    const notification = new Notification(title, options);
    
    // Handle notification click
    notification.onclick = () => {
        console.log('Notification clicked');
        window.focus();
        notification.close();
    };
    
    // Automatically close after 5 seconds
    setTimeout(() => {
        notification.close();
    }, 5000);
    
    return notification;
}

// Example usage
async function demoNotification() {
    const hasPermission = await requestNotificationPermission();
    
    if (hasPermission) {
        showNotification('Hello!', {
            body: 'This is a sample notification from the Browser APIs Explorer.',
            icon: 'https://example.com/icon.png', // Replace with actual icon URL
            tag: 'sample-notification'
        });
    }
}

Fetch API

The Fetch API provides a modern interface for making HTTP requests. It is more powerful and flexible than the older XMLHttpRequest and uses Promises for better async handling.

The fetch() method takes a URL and returns a Promise that resolves to the Response object representing the response to the request.

Example: Making HTTP Requests with Fetch

// Basic GET request
async function fetchData(url) {
    try {
        const response = await fetch(url);
        
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }
        
        const data = await response.json();
        return data;
    } catch (error) {
        console.error('Error fetching data:', error);
    }
}

// POST request with JSON data
async function postData(url, data) {
    try {
        const response = await fetch(url, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
            },
            body: JSON.stringify(data),
        });
        
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }
        
        const responseData = await response.json();
        return responseData;
    } catch (error) {
        console.error('Error posting data:', error);
    }
}

// Example usage
async function demonstrateFetch() {
    // Fetch data from a public API
    const userData = await fetchData('https://jsonplaceholder.typicode.com/users/1');
    console.log('User data:', userData);
    
    // Post data to a test API
    const postResult = await postData('https://jsonplaceholder.typicode.com/posts', {
        title: 'Test Post',
        body: 'This is a test post using fetch()',
        userId: 1,
    });
    console.log('Post result:', postResult);
}

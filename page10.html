Debugging Tools

Modern browsers come with powerful debugging tools that help developers identify and fix issues in their JavaScript code. The most commonly used tool is the browser's Developer Tools (DevTools), which includes a JavaScript debugger.

Key Features

  • Breakpoints: Pause code execution at specific lines
  • Step through code: Execute code line by line
  • Watch expressions: Monitor values of variables and expressions
  • Call stack: View the sequence of function calls
  • Scope inspection: Examine variables in different scopes

Using Debugger Statement

function calculateTotal(price, quantity) {
    // This will pause execution if DevTools is open
    debugger;
    
    const subtotal = price * quantity;
    const tax = subtotal * 0.08;
    const total = subtotal + tax;
    
    return total;
}

// Test the function
console.log("Total: $" + calculateTotal(25, 3));

Setting Breakpoints Programmatically

function complexCalculation(values) {
    if (values.length === 0) {
        console.error("No values provided");
        return;
    }
    
    // Conditional breakpoint - only pauses if condition is true
    if (values.some(v => v < 0)) {
        debugger; // Pause if any value is negative
    }
    
    const result = values.reduce((acc, val) => {
        return acc + (val * 2);
    }, 0);
    
    return result / values.length;
}

// Test with different inputs
console.log("Result 1:", complexCalculation([1, 2, 3, 4, 5]));
console.log("Result 2:", complexCalculation([-1, 2, 3])); // This will trigger the debugger

Console Methods

The console object provides several methods for logging information, debugging code, and measuring performance. These methods are essential for understanding what's happening in your code.

Common Console Methods

  • console.log(): Outputs a message to the console
  • console.error(): Outputs an error message
  • console.warn(): Outputs a warning message
  • console.table(): Displays data as a table
  • console.time() & console.timeEnd(): Measures time between operations
  • console.group() & console.groupEnd(): Groups related messages

Basic Console Methods

// Warning and error messages

// Informational and debug messages

Error Objects

JavaScript has several built-in error constructors that create specific error objects. Understanding these error types helps in effective debugging and error handling.

Error Types

  • Error: Generic error object
  • SyntaxError: Invalid JavaScript syntax
  • ReferenceError: Reference to non-existent variable
  • TypeError: Value is not of expected type
  • RangeError: Value is outside valid range
  • URIError: Error in encodeURI() or decodeURI()
  • AggregateError: Multiple errors wrapped together

Creating and Throwing Errors

// Creating different types of errors
const genericError = new Error("This is a generic error");
const typeError = new TypeError("Expected a number");
const referenceError = new ReferenceError("Variable is not defined");
const rangeError = new RangeError("Value out of range");

// Log error details
console.log("Generic error:", genericError.message);
console.log("Error stack:", genericError.stack);

// Function that throws errors based on input
function processValue(value) {
    if (typeof value !== "number") {
        throw new TypeError("Value must be a number");
    }
    
    if (value < 0 || value > 100) {
        throw new RangeError("Value must be between 0 and 100");
    }
    
    return value * 2;
}

// Test the function with error handling
try {
    console.log("Result:", processValue(50));
    console.log("Result:", processValue("not a number")); // This will throw
} catch (error) {
    console.error("Caught error:", error.name, "-", error.message);
}

Custom Error Classes

// Creating a custom error class
class ValidationError extends Error {
    constructor(message, field) {
        super(message);
        this.name = "ValidationError";
        this.field = field;
        this.date = new Date();
    }
}

// Using the custom error
function validateUser(user) {
    if (!user.name) {
        throw new ValidationError("Name is required", "name");
    }
    
    if (!user.email || !user.email.includes("@")) {
        throw new ValidationError("Valid email is required", "email");
    }
    
    return true;
}

// Test validation with error handling
try {
    validateUser({ name: "", email: "invalid" });
} catch (error) {
    if (error instanceof ValidationError) {
        console.error(`Validation error in ${error.field}: ${error.message}`);
        console.error("Timestamp:", error.date);
    } else {
        console.error("Unexpected error:", error);
    }
}

Unit Testing (Jest, Mocha)

Unit testing involves testing individual units or components of code in isolation to ensure they work as expected. Jest and Mocha are popular JavaScript testing frameworks.

Key Concepts

  • Test suites: Group of related tests
  • Test cases: Individual tests
  • Assertions: Checking if values match expectations
  • Setup and teardown: Preparing and cleaning up test environment
  • Mocking: Simulating dependencies

Basic Unit Test with Jest

// Simple function to test
function sum(a, b) {
    return a + b;
}

// Test using Jest-style assertions
function testSum() {
    // Test case 1: Positive numbers
    console.assert(sum(1, 2) === 3, "Sum of 1 and 2 should be 3");
    
    // Test case 2: Negative numbers
    console.assert(sum(-1, -2) === -3, "Sum of -1 and -2 should be -3");
    
    // Test case 3: Decimal numbers
    console.assert(sum(1.5, 2.5) === 4, "Sum of 1.5 and 2.5 should be 4");
    
    console.log("All sum tests passed!");
}

// Run the tests
testSum();

Testing Asynchronous Code

// Function that returns a promise
function fetchData() {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve({ data: "Sample data", status: 200 });
        }, 100);
    });
}

// Async test function
async function testFetchData() {
    try {
        const result = await fetchData();
        
        // Assertions
        console.assert(result.status === 200, "Status should be 200");
        console.assert(result.data === "Sample data", "Data should match");
        
        console.log("Async test passed!");
    } catch (error) {
        console.error("Async test failed:", error);
    }
}

// Run the async test

Integration Testing

Integration testing verifies that different modules or services work together correctly. It focuses on testing the interactions between components.

Key Concepts

  • Testing component interactions
  • Verifying data flow between modules
  • Testing API integrations
  • Database integration testing
  • Third-party service integration

API Integration Test Example

// Simulated API functions
const api = {
    getUser: (id) => Promise.resolve({ id, name: "John Doe", email: "john@example.com" }),
    getPosts: (userId) => Promise.resolve([
        { id: 1, title: "Post 1", userId },
        { id: 2, title: "Post 2", userId }
    ])
};

// Integration test: Get user and their posts
async function testUserPostIntegration() {
    try {
        // Get user
        const user = await api.getUser(1);
        console.log("User:", user);
        
        // Get user's posts
        const posts = await api.getPosts(user.id);
        console.log("Posts:", posts);
        
        // Verify integration
        console.assert(user.id === 1, "User ID should be 1");
        console.assert(posts.length > 0, "User should have posts");
        console.assert(posts.every(post => post.userId === user.id), 
                      "All posts should belong to the user");
        
        console.log("Integration test passed!");
    } catch (error) {
        console.error("Integration test failed:", error);
    }
}

// Run the integration test
testUserPostIntegration();

End-to-End Testing

End-to-end (E2E) testing validates the complete flow of an application from start to finish. It simulates real user scenarios and interactions.

Key Concepts

  • Testing complete user workflows
  • Simulating user interactions
  • Testing across multiple systems
  • Validating UI and functionality together
  • Using tools like Cypress, Selenium, or Playwright

E2E Test Simulation

// Simulated user interface functions
const ui = {
    login: (username, password) => {
        console.log(`Logging in with ${username}`);
        return Promise.resolve({ success: true, token: "abc123" });
    },
    navigateToDashboard: () => {
        console.log("Navigating to dashboard");
        return Promise.resolve();
    },
    createPost: (title, content) => {
        console.log(`Creating post: ${title}`);
        return Promise.resolve({ id: 100, title, content });
    },
    logout: () => {
        console.log("Logging out");
        return Promise.resolve();
    }
};

// Simulated E2E test
async function testUserWorkflow() {
    try {
        console.log("Starting E2E test: User creates a post");
        
        // Step 1: Login
        const loginResult = await ui.login("testuser", "password");
        console.assert(loginResult.success, "Login should succeed");
        console.assert(loginResult.token, "Should receive auth token");
        
        // Step 2: Navigate to dashboard
        await ui.navigateToDashboard();
        
        // Step 3: Create a post
        const post = await ui.createPost("Test Post", "This is test content");
        console.assert(post.id, "Post should have an ID");
        console.assert(post.title === "Test Post", "Post title should match");
        
        // Step 4: Logout
        await ui.logout();
        
        console.log("E2E test completed successfully!");
    } catch (error) {
        console.error("E2E test failed:", error);
    }
}

// Run the E2E test
testUserWorkflow();

Test-Driven Development

Test-Driven Development (TDD) is a software development approach where tests are written before the actual code. The process follows a cycle: Red, Green, Refactor.

TDD Cycle

  • Red: Write a failing test
  • Green: Write minimal code to pass the test
  • Refactor: Improve code while keeping tests passing
  • Repeat: Continue the cycle for each feature

TDD Example: String Calculator

// Step 1: Write a failing test (RED)
function testStringCalculator() {
    // Test empty string returns 0
    try {
        const result = add("");
        console.assert(result === 0, `Empty string should return 0, got ${result}`);
        console.log("✓ Empty string test passed");
    } catch (e) {
        console.log("✗ Empty string test failed:", e.message);
    }
    
    // Test single number returns the number
    try {
        const result = add("5");
        console.assert(result === 5, `Single number should return itself, got ${result}`);
        console.log("✓ Single number test passed");
    } catch (e) {
        console.log("✗ Single number test failed:", e.message);
    }
}

// Step 2: Write minimal implementation (GREEN)
function add(numbers) {
    if (numbers === "") return 0;
    return parseInt(numbers);
}

// Run the tests
console.log("Running TDD tests for string calculator:");
testStringCalculator();

// Step 3: Refactor and add more tests (REFACTOR)

Mocking & Stubbing

Mocking and stubbing are techniques used in testing to replace real dependencies with controlled substitutes. This allows for isolated testing of components.

Key Concepts

  • Mocks: Objects that simulate the behavior of real objects
  • Stubs: Replace functions with controlled versions
  • Spies: Record information about function calls
  • Dependency injection: Providing dependencies externally

Mocking API Calls

// Real API function (would normally make HTTP request)
function fetchUserData(userId) {
    // This would make an actual API call in production
    return Promise.resolve({ id: userId, name: "Real User" });
}

// Function we want to test
async function getUserName(userId) {
    const user = await fetchUserData(userId);
    return user.name;
}

// Test with a mock
async function testGetUserName() {
    // Save original function
    const originalFetch = fetchUserData;
    
    // Create mock
    fetchUserData = async (userId) => {
        console.log(`Mock called with userId: ${userId}`);
        return { id: userId, name: "Mock User" };
    };
    
    try {
        // Test with mock
        const name = await getUserName(123);
        console.assert(name === "Mock User", `Expected "Mock User", got "${name}"`);
        console.log("✓ Mock test passed");
    } catch (error) {
        console.error("Mock test failed:", error);
    } finally {
        // Restore original function
        fetchUserData = originalFetch;
    }
}

// Run the test
testGetUserName();

// Test with original function
getUserName(456).then(name => {
    console.log("Original function result:", name);
});

Performance Testing

Performance testing measures the speed, responsiveness, and stability of an application under various conditions. It helps identify bottlenecks and optimize code.

Key Concepts

  • Load testing: Testing under expected load
  • Stress testing: Testing beyond normal capacity
  • Benchmarking: Comparing performance metrics
  • Profiling: Identifying performance bottlenecks

Basic Performance Measurement

// Function to test performance of
function heavyComputation(n) {
    let result = 0;
    for (let i = 0; i < n; i++) {
        for (let j = 0; j < n; j++) {
            result += i * j;
        }
    }
    return result;
}

// Performance test function
function testPerformance() {
    const sizes = [100, 200, 400];
    const results = [];
    
    for (const size of sizes) {
        // Start timer
        const startTime = performance.now();
        
        // Run the function
        heavyComputation(size);
        
        // End timer
        const endTime = performance.now();
        const duration = endTime - startTime;
        
        results.push({ size, duration: duration.toFixed(2) });
    }
    
    // Display results
    console.table(results);
    
    // Analyze results
    console.log("Performance analysis:");
    for (let i = 1; i < results.length; i++) {
        const prev = results[i-1];
        const curr = results[i];
        const factor = (curr.size / prev.size).toFixed(1);
        const timeIncrease = (curr.duration / prev.duration).toFixed(1);
        console.log(`Size ${factor}x larger, time ${timeIncrease}x longer`);
    }
}

// Run performance test
testPerformance();

Browser DevTools

Browser Developer Tools are essential for debugging, profiling, and optimizing web applications. They provide insights into HTML, CSS, JavaScript, network activity, and performance.

Key DevTools Features

  • Elements panel: Inspect and modify HTML/CSS
  • Console: Execute JavaScript and view logs
  • Sources: Debug JavaScript with breakpoints
  • Network: Monitor network requests
  • Performance: Analyze runtime performance
  • Memory: Identify memory leaks

Using Console API for Debugging

    console.time("Expensive operation");

    console.timeEnd("Expensive operation");
